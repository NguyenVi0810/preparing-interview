## Entity Relationships

<details>
  <summary>Relationship Types Overview</summary>
  <br/>

  JPA supports four types of relationships between entities.

  **Relationship Types:**

  ```
  1. @OneToOne   - One entity relates to one other entity
  2. @OneToMany  - One entity relates to many other entities
  3. @ManyToOne  - Many entities relate to one entity
  4. @ManyToMany - Many entities relate to many other entities
  ```

  **Directionality:**
  + **Unidirectional**: Only one side knows about the relationship
  + **Bidirectional**: Both sides know about the relationship

  **Ownership:**
  + **Owning side**: Contains the foreign key (uses @JoinColumn)
  + **Inverse side**: References the owning side (uses mappedBy)

</details>

<details>
  <summary>@OneToOne Relationship</summary>
  <br/>

  One entity instance relates to exactly one instance of another entity.

  **Example: User ↔ Profile (one user has one profile)**

  **Unidirectional @OneToOne:**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      
      @OneToOne
      @JoinColumn(name = "profile_id")  // Foreign key column
      private Profile profile;
  }
  
  @Entity
  public class Profile {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String bio;
      private String avatar;
  }
  ```

  **Bidirectional @OneToOne:**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      
      @OneToOne
      @JoinColumn(name = "profile_id")  // Owning side
      private Profile profile;
  }
  
  @Entity
  public class Profile {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String bio;
      
      @OneToOne(mappedBy = "profile")  // Inverse side
      private User user;
  }
  ```

  **Usage:**

  ```java
  @Service
  public class UserService {
      
      @Autowired
      private UserRepository userRepository;
      
      @Transactional
      public void createUserWithProfile() {
          Profile profile = new Profile();
          profile.setBio("Software Developer");
          
          User user = new User();
          user.setUsername("john");
          user.setProfile(profile);
          
          userRepository.save(user);
      }
  }
  ```

  **@OneToOne with shared primary key:**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      
      @OneToOne(mappedBy = "user", cascade = CascadeType.ALL)
      private Profile profile;
  }
  
  @Entity
  public class Profile {
      
      @Id
      private Long id;  // Same as User ID
      
      private String bio;
      
      @OneToOne
      @MapsId  // Use User's ID as Profile's ID
      @JoinColumn(name = "id")
      private User user;
  }
  ```

</details>

<details>
  <summary>@ManyToOne Relationship</summary>
  <br/>

  Many entity instances relate to one instance of another entity.

  **Example: Order → User (many orders belong to one user)**

  **Unidirectional @ManyToOne:**

  ```java
  @Entity
  public class Order {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private BigDecimal total;
      
      @ManyToOne
      @JoinColumn(name = "user_id")  // Foreign key
      private User user;
  }
  
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      // No reference to orders
  }
  ```

  **Usage:**

  ```java
  @Service
  public class OrderService {
      
      @Autowired
      private OrderRepository orderRepository;
      
      @Transactional
      public void createOrder(User user) {
          Order order = new Order();
          order.setTotal(new BigDecimal("99.99"));
          order.setUser(user);
          
          orderRepository.save(order);
      }
  }
  ```

  **@ManyToOne with optional:**

  ```java
  @Entity
  public class Order {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      // Optional relationship (can be null)
      @ManyToOne(optional = true)
      @JoinColumn(name = "user_id")
      private User user;
      
      // Required relationship (cannot be null)
      @ManyToOne(optional = false)
      @JoinColumn(name = "product_id", nullable = false)
      private Product product;
  }
  ```

</details>

<details>
  <summary>@OneToMany Relationship</summary>
  <br/>

  One entity instance relates to many instances of another entity.

  **Example: User → Orders (one user has many orders)**

  **Unidirectional @OneToMany (not recommended):**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      
      @OneToMany
      @JoinColumn(name = "user_id")  // Creates join table
      private List<Order> orders = new ArrayList<>();
  }
  
  @Entity
  public class Order {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private BigDecimal total;
      // No reference to user
  }
  ```

  **Bidirectional @OneToMany (recommended):**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String username;
      
      @OneToMany(mappedBy = "user")  // Inverse side
      private List<Order> orders = new ArrayList<>();
  }
  
  @Entity
  public class Order {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private BigDecimal total;
      
      @ManyToOne
      @JoinColumn(name = "user_id")  // Owning side
      private User user;
  }
  ```

  **Usage:**

  ```java
  @Service
  public class UserService {
      
      @Autowired
      private UserRepository userRepository;
      
      @Transactional
      public void createUserWithOrders() {
          User user = new User();
          user.setUsername("john");
          
          Order order1 = new Order();
          order1.setTotal(new BigDecimal("99.99"));
          order1.setUser(user);
          
          Order order2 = new Order();
          order2.setTotal(new BigDecimal("49.99"));
          order2.setUser(user);
          
          user.getOrders().add(order1);
          user.getOrders().add(order2);
          
          userRepository.save(user);
      }
  }
  ```

  **Helper methods for bidirectional:**

  ```java
  @Entity
  public class User {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, orphanRemoval = true)
      private List<Order> orders = new ArrayList<>();
      
      // Helper method to maintain both sides
      public void addOrder(Order order) {
          orders.add(order);
          order.setUser(this);
      }
      
      public void removeOrder(Order order) {
          orders.remove(order);
          order.setUser(null);
      }
  }
  ```

</details>

<details>
  <summary>@ManyToMany Relationship</summary>
  <br/>

  Many entity instances relate to many instances of another entity.

  **Example: Student ↔ Course (students enroll in many courses, courses have many students)**

  **Unidirectional @ManyToMany:**

  ```java
  @Entity
  public class Student {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String name;
      
      @ManyToMany
      @JoinTable(
          name = "student_course",  // Join table name
          joinColumns = @JoinColumn(name = "student_id"),  // FK to Student
          inverseJoinColumns = @JoinColumn(name = "course_id")  // FK to Course
      )
      private List<Course> courses = new ArrayList<>();
  }
  
  @Entity
  public class Course {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String name;
      // No reference to students
  }
  ```

  **Bidirectional @ManyToMany:**

  ```java
  @Entity
  public class Student {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String name;
      
      @ManyToMany
      @JoinTable(
          name = "student_course",
          joinColumns = @JoinColumn(name = "student_id"),
          inverseJoinColumns = @JoinColumn(name = "course_id")
      )
      private List<Course> courses = new ArrayList<>();
      
      // Helper methods
      public void enrollCourse(Course course) {
          courses.add(course);
          course.getStudents().add(this);
      }
      
      public void dropCourse(Course course) {
          courses.remove(course);
          course.getStudents().remove(this);
      }
  }
  
  @Entity
  public class Course {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      private String name;
      
      @ManyToMany(mappedBy = "courses")  // Inverse side
      private List<Student> students = new ArrayList<>();
  }
  ```

  **Usage:**

  ```java
  @Service
  public class StudentService {
      
      @Autowired
      private StudentRepository studentRepository;
      
      @Transactional
      public void enrollStudent(Student student, Course course) {
          student.enrollCourse(course);
          studentRepository.save(student);
      }
  }
  ```

  **@ManyToMany with extra columns (use intermediate entity):**

  ```java
  // Instead of @ManyToMany, create intermediate entity
  @Entity
  public class Enrollment {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      @ManyToOne
      @JoinColumn(name = "student_id")
      private Student student;
      
      @ManyToOne
      @JoinColumn(name = "course_id")
      private Course course;
      
      private LocalDateTime enrolledAt;
      private String grade;
  }
  
  @Entity
  public class Student {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      @OneToMany(mappedBy = "student")
      private List<Enrollment> enrollments = new ArrayList<>();
  }
  
  @Entity
  public class Course {
      
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;
      
      @OneToMany(mappedBy = "course")
      private List<Enrollment> enrollments = new ArrayList<>();
  }
  ```

</details>

<details>
  <summary>Fetch Types: LAZY vs EAGER</summary>
  <br/>

  Fetch type determines when related entities are loaded.

  **LAZY (default for @OneToMany, @ManyToMany):**

  ```java
  @Entity
  public class User {
      
      @Id
      private Long id;
      
      @OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
      private List<Order> orders;  // Not loaded until accessed
  }
  
  // Usage
  User user = userRepository.findById(1L).get();
  // SELECT * FROM users WHERE id = 1
  
  user.getOrders().size();  // Triggers second query
  // SELECT * FROM orders WHERE user_id = 1
  ```

  **EAGER (default for @ManyToOne, @OneToOne):**

  ```java
  @Entity
  public class Order {
      
      @Id
      private Long id;
      
      @ManyToOne(fetch = FetchType.EAGER)
      private User user;  // Loaded immediately
  }
  
  // Usage
  Order order = orderRepository.findById(1L).get();
  // SELECT * FROM orders o LEFT JOIN users u ON o.user_id = u.id WHERE o.id = 1
  ```

  **Best practices:**

  ```java
  // ✅ DO: Use LAZY by default
  @OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
  private List<Order> orders;
  
  // ✅ DO: Use fetch join when you need related entities
  @Query("SELECT u FROM User u LEFT JOIN FETCH u.orders WHERE u.id = :id")
  User findByIdWithOrders(@Param("id") Long id);
  
  // ❌ DON'T: Use EAGER for collections
  @OneToMany(fetch = FetchType.EAGER)  // Can cause performance issues
  private List<Order> orders;
  ```

</details>

<details>
  <summary>Cascade Types</summary>
  <br/>

  Cascade operations propagate from parent to child entities.

  **Cascade types:**

  ```java
  @Entity
  public class User {
      
      @Id
      private Long id;
      
      // CascadeType.PERSIST - save children when saving parent
      @OneToMany(mappedBy = "user", cascade = CascadeType.PERSIST)
      private List<Order> orders1;
      
      // CascadeType.MERGE - merge children when merging parent
      @OneToMany(mappedBy = "user", cascade = CascadeType.MERGE)
      private List<Order> orders2;
      
      // CascadeType.REMOVE - delete children when deleting parent
      @OneToMany(mappedBy = "user", cascade = CascadeType.REMOVE)
      private List<Order> orders3;
      
      // CascadeType.REFRESH - refresh children when refreshing parent
      @OneToMany(mappedBy = "user", cascade = CascadeType.REFRESH)
      private List<Order> orders4;
      
      // CascadeType.DETACH - detach children when detaching parent
      @OneToMany(mappedBy = "user", cascade = CascadeType.DETACH)
      private List<Order> orders5;
      
      // CascadeType.ALL - all cascade operations
      @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
      private List<Order> orders6;
      
      // Multiple cascade types
      @OneToMany(mappedBy = "user", cascade = {CascadeType.PERSIST, CascadeType.MERGE})
      private List<Order> orders7;
  }
  ```

  **CascadeType.PERSIST example:**

  ```java
  @Entity
  public class User {
      
      @OneToMany(mappedBy = "user", cascade = CascadeType.PERSIST)
      private List<Order> orders = new ArrayList<>();
  }
  
  @Transactional
  public void createUserWithOrders() {
      User user = new User("john");
      
      Order order1 = new Order(new BigDecimal("99.99"));
      Order order2 = new Order(new BigDecimal("49.99"));
      
      user.addOrder(order1);
      user.addOrder(order2);
      
      userRepository.save(user);  // Saves user AND orders
  }
  ```

  **CascadeType.REMOVE example:**

  ```java
  @Entity
  public class User {
      
      @OneToMany(mappedBy = "user", cascade = CascadeType.REMOVE)
      private List<Order> orders = new ArrayList<>();
  }
  
  @Transactional
  public void deleteUser(Long userId) {
      User user = userRepository.findById(userId).orElseThrow();
      userRepository.delete(user);  // Deletes user AND orders
  }
  ```

</details>

<details>
  <summary>orphanRemoval</summary>
  <br/>

  `orphanRemoval` deletes child entities when removed from parent collection.

  **Without orphanRemoval:**

  ```java
  @Entity
  public class User {
      
      @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
      private List<Order> orders = new ArrayList<>();
  }
  
  @Transactional
  public void removeOrder() {
      User user = userRepository.findById(1L).orElseThrow();
      user.getOrders().remove(0);  // Order still exists in database!
      userRepository.save(user);
  }
  ```

  **With orphanRemoval:**

  ```java
  @Entity
  public class User {
      
      @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, orphanRemoval = true)
      private List<Order> orders = new ArrayList<>();
  }
  
  @Transactional
  public void removeOrder() {
      User user = userRepository.findById(1L).orElseThrow();
      user.getOrders().remove(0);  // Order is deleted from database
      userRepository.save(user);
  }
  ```

  **Difference between CascadeType.REMOVE and orphanRemoval:**

  ```java
  // CascadeType.REMOVE - deletes children when parent is deleted
  @OneToMany(cascade = CascadeType.REMOVE)
  private List<Order> orders;
  
  userRepository.delete(user);  // Deletes user and orders
  user.getOrders().remove(0);   // Order NOT deleted
  
  // orphanRemoval - deletes children when removed from collection
  @OneToMany(orphanRemoval = true)
  private List<Order> orders;
  
  userRepository.delete(user);  // Deletes user, orders remain
  user.getOrders().remove(0);   // Order is deleted
  
  // Both - deletes in both cases
  @OneToMany(cascade = CascadeType.ALL, orphanRemoval = true)
  private List<Order> orders;
  ```

</details>

<details>
  <summary>Best Practices</summary>
  <br/>

  ```java
  // ✅ DO: Use bidirectional @OneToMany with @ManyToOne
  @Entity
  public class User {
      @OneToMany(mappedBy = "user")
      private List<Order> orders;
  }
  
  @Entity
  public class Order {
      @ManyToOne
      @JoinColumn(name = "user_id")
      private User user;
  }
  
  // ❌ DON'T: Use unidirectional @OneToMany
  @Entity
  public class User {
      @OneToMany
      @JoinColumn(name = "user_id")
      private List<Order> orders;  // Creates extra queries
  }
  
  // ✅ DO: Use LAZY fetch by default
  @OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
  private List<Order> orders;
  
  // ❌ DON'T: Use EAGER for collections
  @OneToMany(fetch = FetchType.EAGER)
  private List<Order> orders;  // Performance issues
  
  // ✅ DO: Initialize collections
  @OneToMany(mappedBy = "user")
  private List<Order> orders = new ArrayList<>();
  
  // ❌ DON'T: Leave collections null
  @OneToMany(mappedBy = "user")
  private List<Order> orders;  // NullPointerException
  
  // ✅ DO: Use helper methods for bidirectional relationships
  public void addOrder(Order order) {
      orders.add(order);
      order.setUser(this);
  }
  
  // ✅ DO: Use orphanRemoval for parent-child relationships
  @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, orphanRemoval = true)
  private List<Order> orders;
  
  // ✅ DO: Use intermediate entity for @ManyToMany with extra columns
  @Entity
  public class Enrollment {
      @ManyToOne
      private Student student;
      
      @ManyToOne
      private Course course;
      
      private LocalDateTime enrolledAt;
  }
  
  // ❌ DON'T: Use @ManyToMany with extra columns
  @ManyToMany
  private List<Course> courses;  // Cannot add enrolledAt
  
  // ✅ DO: Use @JoinColumn on @ManyToOne side
  @ManyToOne
  @JoinColumn(name = "user_id")
  private User user;
  
  // ✅ DO: Use mappedBy on @OneToMany side
  @OneToMany(mappedBy = "user")
  private List<Order> orders;
  
  // ✅ DO: Override equals() and hashCode() for entities in collections
  @Override
  public boolean equals(Object o) {
      if (this == o) return true;
      if (!(o instanceof Order)) return false;
      Order order = (Order) o;
      return id != null && id.equals(order.id);
  }
  
  @Override
  public int hashCode() {
      return getClass().hashCode();
  }
  ```

  **Key Points**
  <br/>

  + **@OneToOne**: One entity relates to one other entity
  + **@ManyToOne**: Many entities relate to one entity (owning side)
  + **@OneToMany**: One entity relates to many entities (inverse side)
  + **@ManyToMany**: Many entities relate to many entities
  + **Bidirectional** relationships are preferred over unidirectional
  + **Owning side** contains foreign key (@JoinColumn)
  + **Inverse side** uses mappedBy attribute
  + Use **LAZY** fetch by default for collections
  + Use **EAGER** only when always needed (default for @ManyToOne, @OneToOne)
  + **CascadeType.PERSIST**: Save children when saving parent
  + **CascadeType.MERGE**: Merge children when merging parent
  + **CascadeType.REMOVE**: Delete children when deleting parent
  + **CascadeType.ALL**: All cascade operations
  + **orphanRemoval**: Delete children when removed from collection
  + Always **initialize collections** (= new ArrayList<>())
  + Use **helper methods** to maintain both sides of bidirectional relationships
  + Use **intermediate entity** for @ManyToMany with extra columns
  + Override **equals() and hashCode()** for entities in collections
  + Use **@JoinTable** for @ManyToMany relationships
  + Use **fetch joins** to avoid N+1 problem with LAZY loading

</details>
